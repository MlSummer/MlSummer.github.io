REDIS缓存

Nosql概述

NoSQL = Not Only SQL (不仅仅是SQL)

泛指非关系型数据库的，随着web2.0 传统的关系型数据库 很难对付web2.0时代

特点

  1. 方便扩展（数据之间没有关系，很好扩展！）

  2. 大数据量高性能（Redis一秒写8万次。读取11万，NoSql的缓存记录级，是一种细粒度的缓存，性能会比较高）

  3. 数据类型的多样性（不需要事先设计数据库）

  4. 传统RDBMS和NoSQL

四大类型

键值对存储 Redis

列存储 HBase

文档存储 MongoDB

图形数据库（社交关系）

Redis入门

Remote Dictionary Server 远程字典服务

开源 C语言编写 支持网络 key-value

当下最热门的NoSQL技术

  1. 内存存储 持久化
  2. 效率高
  3. 发布订阅
  4. 地图信息
  5. 计时器

基础知识

五大数据类型

Redis-Key

select index 切换数据库

DBSIZE 数据库长度

keys * 查看数据库所有key

flushdb 清空当前数据库

flushall 清楚全部数据库的内容

EXISTS key 判断某个键是否存在（存在返回1 不存在返回0）

MOVE key 移除某一个key

EXPIRE key time 设置某一个key time秒后过期

ttl key 查看当前key的剩余时间

type name 查看当前key的一个类型

String（字符串）

APPEND key "data" 在key对应value字符串追加内容

incr key 某一个key数据自增1

incrby key i 某一个key数据自增i

decr key 某一个key数据自减1

decrby key i 某一个key数据自减i

GETRANGE key 0 3 截取字符串[0,3]

GETRANGE key 0 -1 截取全部字符串

SETRANGE key 1 替换指定位置开始的字符串

setex key time value 设置过期时间

setnx key value 如果key不存在 创建成功，否则失败

mget k1 k2 k3 同时获取多个值

msetnx k1 v1 k2 v2 同时设置多个值（原子性操作，同时成功失败）

set user:1 {name:xdj,age:3} 设置一个对象

List（列表）

LPUSH list value 在list左边插值

RPUSH list value 在list右边插值

LPOP list 移除列表第一个元素

RPOP list 移除列表最后一个元素

LRANGE LIST 0 -1 获取list数据 从左到右

RRANGE LIST 0 -1 获取list数据 从右到左

LREM list i value 移除list中指定i个value

ltrim list 1 2 通过下标截取指定下标间的内容

rpoplpush list list2 移除list中最后一个元素到list2中

Set（集合）

sadd list value 在set中存入一个值

SMEMBERS list 查看指定set中所有值

scard list 获取set集合中元素个数

srem list value 移除set集合中指定元素

SRANDMEMBER list 随机获取set中一个元素

SDIFF key1 key2 差集

SINTER key1 key2 交集

SUNION key1 key2 并集

Hash（哈希）

Map集合 key-map

hset hash field xiadujian 在hash中添加一个map key为field value为xiadujian

hget hash field 获取关键字field对应的map

hmset hash field xiadujian field1 xiatiantian 添加多个值

hmget hash field field1 获取多个值

hgetall hash 获取所有

hdel hash field 删除指定的key对应值

Zset（有序集合）

zadd set i value 添加一个值

zadd set 1 value1 2 value2 添加多个值

ZRANGEBYSCORE salary min max 排序 从最小值到最大值（-inf +inf 负无穷 正无穷）

三种特殊数据类型

geospatial地理位置

geoadd china:city [经度] [纬度] key 添加一个地理位置

geopos china:city key 获取指定超市的经度纬度

geodist china city key1 key2 [单位] 返回给定两个坐标距离 单位 km,m,mi(英尺),ft(英里)

georadius china:city [经度][纬度] value km withdist查询半径value以内的所有坐标(withdist:返回形式位置+距离）

georadius china:city [经度][纬度] value km withcoord查询半径value以内的所有坐标(withdist:返回形式位置+经纬度）

Hyperloglog基数统计

优点相比于set计数所占内存极少

PFadd key value 添加

PFCOUNT key 计数

PFMERGE key3 key2 key1 12=3合并

Bitmaps

位存储 一般用于状态 是否

setbit sign key value 添加

事务

事务本质：一组命令的集合

Redis单条命令保持原子性，但是事务不保证原子性

Redis事务没有隔离级别的概念 如脏读…

redis事务命令：

  • 开启事务（multi）
  • 命令入队（所有需要执行的命令…）
  • 执行事务（exec）
  • 放弃事务（DISCARD）

监控：Watch

watch 监视

unwatch 取消监视

集成SpringBoot

在SpringBoot2.X之后，原来使用的jedis被替换为了lettuce

jedis:采用的直连，多个线程操作不安全 使用jedis pool连接池

lettuce:采用netty,实例可以再多个线程中进行共享，不存在线程不安全

持久化 RDB

（Redis DataBase）快照 默认配置足以 在指定的时间间隔内将内存中的数据集快照写入磁盘 效率较高 保存的文件是dump.rdb

save ss number在配置文件中设置ss秒内修改number个数据进行保存

通过添加子进程额外进行数据

优：适合大规模的数据恢复 对数据完整性要求不高

缺：fork进程的时候会占用一定的内容空间 需要一定的时间间隔进行操作，遇到redis意外宕机 当此修改就无效了

持久化 AOF

（Append Only File）

以日志的方式记录每个写操作（除读取操作），只追加文件，但不能改写文件，redis启动时会读取该文件进行数据的重构

默认是不开启的，需要手动配置

appendonly no 在配置文件中修改appendonly 只需将no改为 yes

优：每次修改同步，文件完整性好 只可能丢失1s的数据 从不同步效率最高

缺：相对于数据文件 AOF远大于RDB 修复速度慢 运行效率也较慢

数据的复制是单向的，只能从主–>从 读写分离 80%是读操作 一般是一主二从

作用 数据冗余 故障恢复 负载均衡 高可用
